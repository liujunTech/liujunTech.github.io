Java

第三章：面向对象

一丶构造方法

1.方法名和类名必须保持一致

2.无返回值类型，连void都没有

3.通过new来调用

4.无参构造器问题：没有手动定义构造器，系统会自动添加一个无参的构造器。如果自己定义了构造器，系统就不会自动添加。

5.构造方法的第一句总是super，直接调用父类的构造方法。

二丶方法的重载（overload)

两同（同一个类，同一方法名） 三不同（参数列表不同，类型个数，顺序不同）

返回值不同，构成重载吗？ 否

形参名称不同，构成重载吗？否

关键字：this：普通方法中，调用本方法的对象。构造方法中，正要初始化的对象。

还可以调用其他的构造方法

super：指向直接父类的引用

static：用它修饰的变量和方法，就变成静态变量和静态方法。从属于类，通过类名调用，实际存储于方法区中

package：非注释行的第一句。

impor：引入外部的类。import static 导入包的属性

final：修饰变量：常量（命名规范：全部大写，多个单词下划线隔开）

修饰方法：不能被重写 修饰类：不能被继承。

三丶面向对象的三大特征：

封装：通过private，default，protected，public关键字实现属性或方法封装。“高内聚低耦合”

继承： 通过extends。优点：提高代码的重用、通过继承对现实世界更好的建模

注意：Java中类只有单继承，没有c++的多继承

四丶方法的重写：

等于：方法名保持一致

大于等于：子类的修饰符可以大于等于父类的

小于等于：子类的返回值类型小于父类的类型。子类声明的异常类型不能超过父类

Object类

1、所有类的根基类

2、toSpring（打印对象）

3、equals、hashcode

4、wait、notify、notifyAll

多态：三个必要条件：继承、方法的重写、父类引用指向子类对象

动态绑定；重载的方法

静态绑定：重写的方法

抽象类

1、包含抽象房方法的类一定是抽象类

2、抽象类不能被new

3、抽象类可以包含：普通方法、成员变量、构造方法

注意：普通变量不能被抽象类修饰

接口

1、interface

2、类实现接口：implements 可以实现多个接口

3、接口可以多继承

4、接口定义的一组规范

内存机制

栈：存放局部变量，不可以被多个线程共享。空间连续、速度快

堆：存放对象，可以被多个线程共享 。空间不连续、速度慢、但是灵活

方法区：存放类的信息、代码、静态变量、字符串常量等。可以被多个线程共享。空间不连续、速度慢、但是灵活。

垃圾回收器（GC Garbage Collection)

1、程序员不能去调用垃圾回收机制。但是可以通过System.gc()建议回收。

2、finalize：一般也不用它调

递归算法：

1、递归头：什么时候不调用用自己

2、递归题：什么时候调用自己

第四章：异常

异常机制：

1、try catch （先小后大）finally

2、声明抛出异常：throws

3、手动抛出异常：throw

4、自定义异常：extends Exception 或者他的子类

数组

1、长度固定。不可变

2、所有元素类型一致

3、元素类型是任意类型

4、通过下标引用相关元素，下标不能超过length-1

5、数组也是对象，数组的元素可以看成为对象的成员变量。

6、多维数组：数组的元素是数组

7、数组的三种初始化方式：

动态初始化

静态初始化： int[] a {1,2,3}

默认初始化

8、Arrays类：包含了对数据元素的排序、查找

9、算法：冒泡排序 、二分法

第五章：容器

容器：

1、Collection

​ a）list（有序可重复）：

ArrayLIst：底层用数组实现，线程不安全，效率高

LinkedList：底层用双层链表实现，线程不安全，效率高

Vector：底层用数组实现，线程安全，效率低

​ b）set(无序，不可重复)

Hashset：内部使用HashMap实现

2、Map

采取“key-value”来存储我们的数据。

a）HashMap线程不安全，效率高

b）HashTable线程安全效率低

3、iterator

通过它实现遍历容器中的元素

4、泛型

5、Collection工具类

常用类

1.包装类

2、String/stringBuilder/StringBuffer

String：不可变字符序列

StringBuilder：可变字符序列，线程不安全

StringBuffer：可变字符序列、线程安全

3、时间：Date、DateFormat、Calendar

Date：使用毫秒数表示时间概念。

DateFormat：实现Date对象跟 String对象的互相转（根据指定格式互相转化）

Calendar：日期类。实现了将人的日期概念和Date的互相转化

4、File

代表文件或者目录的一个抽象

5、Math

数学类

6、枚举

如果定义一组常量，建议使用枚举。

第七章：多线程

1、定义多线程的两种方式：

a）Extends Thread

b）Implements Runnable

2、线程的状态：

a）new

b）就绪状态（调用start方法后）

c）运行状态

d）阻塞状态

1） sleep：不会释放持有的对象锁

2）join：等待另一个线程执行完成

3）wait：会释放持有的对象锁

4）synchronized：运行同步代码，但是没有获得制定独对象的锁，于是进入指定对象的锁池等待

e）终止

1）不建议使用：stop destory直接终止

2）建议通过boolean变量，正常终止线程运行

3、线程的优先级：

a）1-10，默认是5

b）优先级低只是意味着调用的概率低，并不是不被调用。

4、资源同步：

Synchronized 修饰方法、语句块

5、死锁

建议不要同时持有多个对象的锁

6、“生产者-消费者”模式(解决资源冲突的问题)

2、InterSocketAddress：封装了IP和端口。

3、URL：代表一个网络的资源

a）openStream

4、TCP通讯

a)特点：面向连接、安全、稳定。

b）ServerSocket

c）Socket

a）特点：不面向连接，可能会丢数据，发送数据效率高。

b）DatagramPacket：数据打包

c）DatagramSocket：数据发送和接收
